# 美食偏好管理 UI 重设计方案

## 一、设计分析

### 参考图关键特征
1. **手绘铅笔线条边框** — 卡片边框有不规则毛刺感，不是简单圆角
2. **叶子/藤蔓装饰** — 标题和卡片角落有手绘植物装饰元素
3. **奶油白纸张背景** — 温暖的纸质感
4. **Dashboard 概览** — 统计卡片（大数字+图标+副文案）并排展示
5. **活动流** — 头像+"谁做了什么"+时间 的 feed 布局
6. **Pill chips 带彩色圆点前缀** — 食物标签前有分类色圆点
7. **手绘食物插画装饰** — 番茄、胡萝卜等 CSS 装饰
8. **底部标签栏绿色活跃指示条** — 更粗更明显的活跃条
9. **Invite 按钮手绘虚线边框** — 已有类似实现

### 当前状态
- 已有蜡笔风 CSS（不等圆角、crayon-shadow），但手绘感不够明显
- 缺少：SVG 手绘边框、叶子装饰、统计概览区、活动流样式、彩色圆点前缀
- 单文件 HTML，约 3168 行（CSS ~1043 行，HTML ~330 行，JS ~1800 行）

## 二、改造方案

### A. 视觉风格改造（CSS）

1. **手绘边框 SVG filter** — 用 `feTurbulence + feDisplacementMap` 实现真正不规则边框
   - 给 `.category`, `.friend-card`, `.group-card`, `.login-card` 添加 `filter: url(#sketch-edge)` 
   - 调大 displacement scale 从 2.5→4，增强手绘感
   - 给卡片边框增加 `border-image` 或用 `outline` 模拟铅笔线

2. **叶子/藤蔓装饰** — 用 CSS `::before`/`::after` 伪元素 + Unicode/emoji 或内联 SVG
   - 标题区域添加叶子装饰 🌿🍃
   - 分类卡片角落添加小装饰
   - 使用绝对定位的 emoji 装饰元素

3. **手绘食物插画装饰** — 在页面角落和空状态区用 emoji + CSS transform 模拟手绘散落效果

4. **Pill chips 彩色圆点前缀** — 在 `.chip` 前加分类色小圆点
   - 爱吃：薄荷绿圆点
   - 不爱吃：暖黄/橙色圆点
   - 过敏：珊瑚红圆点

5. **底部标签栏** — 活跃指示条改为更粗的绿色条（类似参考图）
   - 指示条从 4px→6px
   - 位置从顶部改为底部

6. **增强纸张纹理** — 给 body 加更明显的纸纹理 noise

7. **卡片阴影增强** — crayon-shadow 加大偏移和模糊度

### B. 页面布局改造（HTML + CSS）

1. **朋友列表页统计概览区** — 在朋友列表顶部添加 3 个并排统计卡片：
   - 📋 总食物数
   - 📂 分类数（3 类各多少）
   - 👥 朋友数
   - 并排展示，移动端 2+1 布局

2. **编辑记录改为活动流** — 修改 `.log-entry` 样式：
   - 增加用户头像（emoji 圆形）
   - "谁 为 谁 做了什么" 叙述格式
   - 时间戳右对齐
   - 添加左侧时间轴线

3. **统计卡片并排** — 使用 CSS Grid 2-3 列布局

### C. 暗色模式适配
- 所有新增元素在 `@media (prefers-color-scheme: dark)` 中适配

## 三、实施步骤

1. **修改 SVG filter** — 增强 sketch-edge filter 参数
2. **新增 CSS 样式** — 叶子装饰、统计卡片、活动流时间轴、芯片圆点、底部标签栏等
3. **修改 HTML 结构** — 添加统计概览区 DOM、修改 log-entry 结构
4. **修改 JS 渲染函数** — `renderFriendList` 中增加统计渲染、`renderEditLogs` 中增加头像和活动流格式
5. **暗色模式** — 适配新增样式
6. **响应式** — 确保三断点（360/480/768）正常

### 不变的部分
- 所有 Supabase 逻辑、CRUD 函数、Realtime 订阅
- 视图切换逻辑
- 导出功能
- 基本 DOM 结构（views、app-wrapper）
