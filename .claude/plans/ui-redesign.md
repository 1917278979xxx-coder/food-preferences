# 美食偏好管理 UI 重设计方案

## 一、设计理解

### 参考图关键视觉特征
1. **手绘铅笔边框**：卡片边框不规则、有毛刺感，真正的 sketch 风
2. **叶子/藤蔓装饰**：标题和卡片角落有手绘植物元素（用 CSS 伪元素模拟）
3. **奶油白纸纹背景**：`#F7F4EC` 为主色，纸张颗粒感
4. **Dashboard 概览区**：统计卡片（大数字 + 图标 + 副文案）并排
5. **活动流**：头像 + "谁做了什么" + 相对时间
6. **Pill chips**：食物标签带彩色圆点前缀
7. **手绘食物插画**：用 emoji 装饰替代（保持纯 CSS/HTML 方案）
8. **底部标签栏**：绿色活跃指示条（已有，强化）
9. **邀请按钮**：手绘虚线边框（已有，强化）

### 当前状态分析
- CSS 已有蜡笔风基础（不等圆角、crayon-shadow、纸纹理）
- 但手绘感不够明显，缺少 SVG filter 真正应用到边框
- 缺少统计概览区
- 编辑记录是弹出面板而非活动流样式
- 无叶子装饰元素
- chips 没有彩色圆点前缀

## 二、改动范围

### CSS 改动（~60%）
1. **手绘边框 SVG filter 增强**
   - 将 `sketch-edge` filter 实际应用到 `.category`、`.friend-card`、`.group-card`
   - 增加手绘风 border-image（用 CSS 生成不规则边框）
   - 更强的 `border-radius` 变化模拟手绘感

2. **叶子/藤蔓装饰**
   - 为 `.header h1`、`.category` 添加 `::before`/`::after` 伪元素
   - 用 CSS 画叶子形状或用 emoji/SVG inline 图标

3. **配色微调**
   - 保持已有配色方案（薄荷绿/暖黄/珊瑚红/浅蓝/奶油白）
   - 增强纸纹背景可见度
   - 加深 crayon-shadow 效果

4. **Pill chips 圆点前缀**
   - `.chip::before` 添加彩色圆点（爱吃=绿、不吃=橙、过敏=红）

5. **底部标签栏绿色指示条**
   - 增强现有 `::before` 伪元素的可见度和宽度

6. **暗色模式同步更新**

### HTML 结构改动（~30%）

1. **朋友列表页 — 添加统计概览区**
   - 在 `friendListView` 的 `.header` 下方添加 `.stats-overview` 区域
   - 三张统计卡片并排：总食物数 / 分类数 / 朋友数
   - 每张卡片：大数字 + 图标 + 副文案

2. **朋友详情页 — 统计概览区**  
   - 在 `detailView` 的 export 按钮上方添加统计概览
   - 三张卡片：爱吃数 / 不吃数 / 过敏数

3. **编辑记录 — 活动流样式**
   - 保留弹出面板（不破坏现有逻辑）
   - 改进 `.log-entry` 样式为更像活动流：头像圆圈 + 文字 + 时间轴线

4. **手绘食物装饰**
   - 在空状态和页头添加食物 emoji 装饰元素

### JS 改动（~10%）
1. `renderFriendList()` — 添加统计概览渲染
2. `renderAll()` — 添加详情页统计概览渲染
3. 新增 `renderStatsOverview()` 函数
4. 所有现有功能逻辑完全保留不动

## 三、不改动的部分
- Supabase 连接和所有 CRUD 操作
- 实时同步逻辑
- 视图切换机制
- 登录/注册流程
- 小组协作功能
- 导出功能
- 头像上传功能
- 筛选/过滤功能

- 朋友详情页添加 `.detail-stats`
- 添加 CSS 样式

### Step 3: 编辑记录活动流改造
- 改进 `.log-entry` 样式为活动流风格
- 添加时间轴线视觉效果

### Step 4: JS 渲染函数
- `renderStatsOverview()` 和 `renderDetailStats()`
- 集成到现有渲染流程

### Step 5: 暗色模式同步
- 所有新增 CSS 变量的暗色模式适配

### Step 6: 响应式适配
- 统计卡片在小屏单列/双列切换
